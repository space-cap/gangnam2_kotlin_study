Jetpack Compose는 Android 네이티브 UI를 빌드하기 위해 Google에서 개발한 최신 선언형 UI 도구 키트입니다. 기존의 XML 기반 뷰 시스템과 달리, 코틀린(Kotlin) 언어만을 사용하여
UI를 구축하므로 개발 과정을 간소화하고 생산성을 높일 수 있습니다.

### 주요 특징

Jetpack Compose는 다음과 같은 주요 특징을 가집니다:

* **선언형 UI**: 개발자는 UI가 '어떻게' 그려져야 하는지가 아닌 '무엇을' 표시할지만을 코드로 선언합니다. 앱의 상태(State)가 변경되면 UI가 자동으로 업데이트되어 코드의 복잡성이 줄어듭니다.
* **더 적은 코드**: 기존의 XML 레이아웃과 Kotlin/Java 코드를 함께 사용하던 방식에 비해 코드의 양이 줄어들고, 전체 버그 발생 가능성도 낮아져 유지보수가 용이합니다.
* **직관적인 Kotlin 구문**: UI 로직을 모두 Kotlin으로 작성하기 때문에, UI와 비즈니스 로직을 하나의 언어로 관리할 수 있어 코드의 가독성과 일관성이 높아집니다. `@Composable`
  어노테이션을 붙인 함수를 사용하여 UI 요소를 정의합니다.
* **개발 속도 향상**: 실시간 미리보기(Live Preview) 기능을 지원하여 코드 변경 사항을 즉시 확인할 수 있습니다. 또한 기존 Android View와의 호환성이 뛰어나 점진적으로 도입할 수 있습니다.
* **강력한 성능**: 머티리얼 디자인, 다크 모드, 애니메이션 등 다양한 최신 Android 기능을 기본적으로 지원하며, 플랫폼 API에 직접 접근할 수 있어 강력한 UI를 구현할 수 있습니다.

### 핵심 개념

Jetpack Compose를 이해하기 위한 몇 가지 핵심 개념은 다음과 같습니다:

* **컴포저블(Composable)**: `@Composable` 어노테이션이 붙은 함수로, UI의 일부를 정의하고 생성합니다. 컴포저블 함수는 다른 컴포저블 함수를 호출하여 복잡한 UI 계층 구조를 만들 수
  있습니다.
* **상태(State)와 리컴포지션(Recomposition)**: `remember`와 `mutableStateOf`를 사용하여 UI의 상태를 관리합니다. 상태가 변경되면 해당 상태를 사용하는 컴포저블 함수만
  다시 실행(리컴포지션)되어 UI를 효율적으로 업데이트합니다.
* **상태 호이스팅(State Hoisting)**: 상태를 하위 컴포저블이 아닌 상위 컴포저블로 이동시켜 관리하는 패턴입니다. 이를 통해 컴포저블의 재사용성을 높이고 상태 관리를 단순화할 수 있습니다.
* **수정자(Modifier)**: UI 요소의 크기, 모양, 동작 등을 꾸미거나 변경하는 데 사용됩니다. 예를 들어 `weight` 수정자는 요소의 공간 분할을 유연하게 처리합니다.
